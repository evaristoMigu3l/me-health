# Architecture Blueprint: Me & Health

This application is a highly comprehensive **React Native + Expo** application designed to serve as a personal health companion. It uses **Expo Router** for file-based navigation, **Zustand** for state management (persisted via `AsyncStorage`), and **NativeWind** (Tailwind CSS) paired with custom `StyleSheet` styling for a rich, modern design.

Below is an in-depth breakdown of how the folders, files, and layers are interconnected.

---

## 1. Data Layer & Models
All data models used to store health metrics, user properties, and app settings are declared in a centralized file:

* [types/index.ts](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/types/index.ts): Declares TypeScript interfaces defining the shape of each health entity in the application:
  * `UserProfile`: User information (name, dob, gender, ethnicity, avatarUri).
  * `Symptom`: Logged symptom data (intensity, duration, location, optional images).
  * `Medication`: Medication logs, dosage units, frequency patterns, custom schedules, and reminders.
  * `Measurement`: Readings (Blood Pressure, Heart Rate, BMI, etc.) and units.
  * `Diagnosis`, `MoodLog`, `DietPlan`, `Investigation` (Scans/Exams), `Activity` (Exercise/Tasks), `SleepLog`, `FoodEntry` (Nutrition), `Document`, `Appointment`, and `HCP` (Healthcare Professionals).
  * `AppSettings`: Settings configurations.

---

## 2. State Management Layer (Stores)
The app uses Zustand stores to hold persisted health, user, authentication, and layout preferences:

* [stores/useHealthStore.ts](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/stores/useHealthStore.ts): Manages all arrays of logged health items (symptoms, activities, sleep logs, etc.). 
  * Integrates with notification triggers (e.g., adding/updating a medication schedules reminders; deleting it cancels them).
  * Persists automatically using `AsyncStorage` under the key `health-storage`.
* [stores/useAuthStore.ts](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/stores/useAuthStore.ts): Manages security logic, PIN locking, and biometric enrollment status (using `expo-secure-store` and `expo-local-authentication`).
  * Features `savePin`, `verifyPin`, `clearPin`, and `authenticateWithBiometric`.
  * Persists under the key `auth-storage`, except for the `isLocked` flag (which is intentionally reset to `true` on cold starts if a PIN exists).
* [stores/useUserStore.ts](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/stores/useUserStore.ts): Stores the user profile configuration and persists under the key `user-storage`.
* [stores/useThemeStore.ts](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/stores/useThemeStore.ts): Manages the current language (English/Portuguese), the theme preference (light/dark/system), and notification alert sounds. Persists under `theme-storage`.

---

## 3. Utility & Core Helpers Layer
* [utils/i18n.ts](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/utils/i18n.ts): Holds the multi-lingual dictionaries supporting English (`en`) and Portuguese (`pt`) for every UI string.
* [utils/notifications.ts](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/utils/notifications.ts): Configures Expo notifications. Computes upcoming dose timings for daily/relative triggers and registers them with the device.
* [utils/exportData.ts](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/utils/exportData.ts): Controls file sharing and generation:
  * Generates clean `.json` file backup formats of the stores.
  * Exports formatted CSV files summarizing all logs.
  * Leverages `expo-print` to transform CSS-styled HTML reports into PDF files and open the OS share sheet.

---

## 4. Custom Hooks Layer
* [hooks/useAppTheme.ts](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/hooks/useAppTheme.ts): Detects current color themes. Extracts light or dark configurations (`colors.background`, `colors.surface`, `colors.primary`, etc.) for clean component consumption.
* [hooks/useTranslation.ts](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/hooks/useTranslation.ts): Exposes the `t` function to render correctly localized labels based on the currently selected language in `useThemeStore`.

---

## 5. UI Components Layer
* [components/LockScreen.tsx](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/components/LockScreen.tsx): The secure keypad input screen for setting up PINs, confirming PIN configurations, and entering PINs/biometrics to unlock the application.
* [components/EditProfileModal.tsx](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/components/EditProfileModal.tsx): An overlay interface enabling profile detail changes.
* `components/ui/` Subfolder: Highly styled UI building blocks:
  * `SearchableDropdown.tsx`: Modal list autocomplete helper to select symptoms, medications, etc.
  * `TrendChart.tsx`: Built with `react-native-gifted-charts` for presenting historical measurement trendlines.
  * `CircularProgressRing.tsx`, `HealthDataCard.tsx`, `ActionCard.tsx`, `CategoryButton.tsx`.

---

## 6. Page Routing Layer (`app/`)
Uses file-based routing to hook screens up:

* [app/_layout.tsx](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/app/_layout.tsx): **Root Entrypoint**. Instantiates the theme provider, coordinates splash screen hide timers, and registers an `AppState` event listener:
  * Lock timing: When the app moves to the background, it starts a timer. If it remains there for more than 5 minutes, it flags `isLocked: true`.
  * Lock overlay: Instead of unmounting the routing tree, the LockScreen is presented as a full-screen Modal overlay. This preserves state and eliminates unmount/remount flickers during unlock transitions.
* `app/(tabs)/` folder: The bottom tab layout configuration:
  * [index.tsx](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/app/(tabs)/index.tsx): Dashboard summarizing today's logged data, shortcuts, medications, and sleep.
  * [track.tsx](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/app/(tabs)/track.tsx): Simple list linking to individual logs or creation forms.
  * [profile.tsx](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/app/(tabs)/profile.tsx): Profile management, PIN/biometric controls, and preferences.
* `app/settings.tsx` [Settings Screen](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/app/settings.tsx): Central screen for units, audio previews, notifications, backups, and file exports.
* Logs & Add forms (e.g. `add-symptom.tsx`, `symptoms-log.tsx`, `symptom-details.tsx`): Screens allowing the user to view lists of past entries, create new items, or edit/delete specific logs.

---

## 7. How the Flows Interconnect (Trace Example)

Let's track what happens when a user **logs a symptom**:
1. The user navigates to `/add-symptom` (which opens [app/add-symptom.tsx](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/app/add-symptom.tsx)).
2. The user types a symptom name (aided by `SearchableDropdown`) and submits the form.
3. The screen dispatches `addSymptom(newSymptom)` to [useHealthStore.ts](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/stores/useHealthStore.ts).
4. The store updates state and calls `AsyncStorage` under the hood to preserve it.
5. The user is redirected back. Both [index.tsx](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/app/(tabs)/index.tsx) and [symptoms-log.tsx](file:///c:/Users/Evaristo/Documents/Development/React-Projects/Codehub_react/Me%20&%20health%20v%202.0/app/symptoms-log.tsx) re-render reactively since they consume the store.

---

## 8. Development & Patch Scripts
In the root directory, there are utility scripts used to fix typical React Native and Expo Router quirks:
* `fix_unmount_flicker.js`: Changes the modal visibility checks from `visible={!!selectedItem}` to a boolean state. This ensures that when the modal is closed, the item isn't immediately set to null, allowing the modal closing animation to finish cleanly without flickering or blanking out.
* `fix_missing_colors.js`: Automatically patches components to ensure they import and invoke `useAppTheme()` instead of referencing missing theme styling structures.
* `disable_modal_animations.js`: Standardizes animation timing across React Native Modals by turning off default transition loops.
